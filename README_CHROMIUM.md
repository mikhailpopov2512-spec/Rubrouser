# Сборка и Настройка «Росбраузера» на базе Chromium для Android

Данный документ описывает процесс автоматизированной компиляции, сборки, подписи и внедрения патчей безопасности в суверенную кодовую базу **Chromium для Android (stable)**.

Благодаря встроенному `Dockerfile` компиляция и развертывание выполняются полностью в изолированном контейнере Docker, избавляя вас от ручной рутинной настройки утилит сборки, JDK, Android NDK/SDK, Clang и Python-интерпретаторов.

---

## 🚀 Три команды для сборки и получения готовых APK

Следуйте этим трем лаконичным шагам для полной автономной компиляции:

### Шаг 1. Сборка Docker-образа компилятора
Данная команда скачает базовый образ, установит Depot Tools, склонирует стабильную ветку Chromium c глубиной коммитов `1` (для максимальной скорости) и применит все наши патчи:
```bash
docker build -t rosbrowser .
```

### Шаг 2. Запуск контейнера компиляции и выгрузка APK
Команда соберет пакеты для архитектур `arm64-v8a` и `armeabi-v7a`, оптимизирует их для дистрибуции без внешних Google-зависимостей и скопирует их обратно на вашу хост-машину:
```bash
docker run --rm -v $(pwd)/out:/output-host rosbrowser
```

### Шаг 3. Заберите готовые файлы
Скомпилированные и оптимизированные бинарные файлы браузера появятся в папке `out` в корневом каталоге вашего проекта:
*   `out/rosbrowser-arm64.apk` — оптимизированный APK под архитектуру `arm64-v8a` (современные смартфоны).
*   `out/rosbrowser-arm.apk` — оптимизированный APK под архитектуру `armeabi-v7a` (более старые устройства).

## 🛠️ Описание примененных суверенных патчей ядра Chromium

Все патчи автоматически применяются на этапе сборки Docker-образа с помощью высокоточных встраиваемых скриптов. Это исключает падения классического `git apply` при несовпадении номеров строк в минорных обновлениях stable-ветки Chromium.

### 1. Государственная символика и флаг РФ
*   **Новая вкладка (NTP Layout)**: В файлы `NewTabPageLayout.java` внедрен метод `dispatchDraw(Canvas canvas)`. Он отрисовывает триколор с идеальными цветами (#FFFFFF, #0039A6, #D52B1E) со специальным светофильтром (#BFFFFFFF / #99000000) для идеальной читаемости поисковой строки и плиток часто посещаемых сайтов. По центру отрисовывается деликатный водяной знак (10% непрозрачности) круглой формы.
*   **Световой индикатор Omnibox**: В `ToolbarPhone.java` переопределен метод `onDraw`. Он визуализирует аккуратную горизонтальную черту шириной 2dp в цветах триколора непосредственно под адресной строкой.

### 2. Безопасность и отсутствие крашей (`RknBlockThrottle`)
Механизм полностью изолирован от главного UI потока:
*   **Кэширование в LruCache**: URL сверяются по быстрому резидентному кэшу `LruCache` со временем доступа менее 1-2 мс.
*   **Асинхронность**: Обращение к спискам за пределами кэша выполняется строго в фоновом потоке `Executor`. Синхронные сети/БД вызовы из методов навигации полностью удалены, что гарантирует защиту от Android ANR (Application Not Responding).
*   **Redirect**: Перенаправление на страницу заглушки `chrome-native://blocked` выполняется строго через Chromium `ThreadUtils.postOnUiThread()`.

### 3. Интеграция Яндекс ID / VK ID Без Сбоев
*   Все сетевые запросы авторизации защищены блоками `try-catch`, предотвращая фатальные исключения.
*   При возврате сервером кода HTTP 401 (токен устарел или аннулирован), приложение не падает, а вызывает `AccountManager.invalidateAuthToken()`, позволяя пользователю безопасно авторизоваться заново.

### 4. Стабильность ГОСТ-криптографии
*   При сборке в `X509Util.java` внедрен метод безопасной динамической инициализации `SSLContext.getInstance("GOST3410")`.
*   При отсутствии загруженной библиотеки `libgost.so` или отсутствии поддержки корневых ГОСТ-сертификатов системой, соединение плавно снижается до стандартного сертифицированного TLS-шифрования без аварийного завершения работы приложения.

## 🔑 Подпись скомпилированных APK перед публикацией

Прежде чем установить собранный APK на реальный Android-смартфон или загрузить его в RuStore, его обязательно необходимо подписать с помощью `apksigner` из Android SDK Build Tools.

1.  **Создайте собственный ключ безопасности (если его еще нет)**:
    ```bash
    keytool -genkey -v -keystore rosbrowser.keystore -alias rosbrowser_alias -keyalg RSA -keysize 2048 -validity 10000
    ```
2.  **Оптимизируйте APK перед подписью (zipalign)**:
    ```bash
    zipalign -v 4 out/rosbrowser-arm64.apk out/rosbrowser-arm64-aligned.apk
    ```
3.  **Подпишите APK вашим ключом**:
    ```bash
    apksigner sign --keystore rosbrowser.keystore --out out/rosbrowser-arm64-signed.apk out/rosbrowser-arm64-aligned.apk
    ```
4.  **Проверьте подпись**:
    ```bash
    apksigner verify out/rosbrowser-arm64-signed.apk
    ```

## 📈 Настройка синхронизации и баз РКН

Для конфигурации серверов синхронизации или списков автоматического обновления реестра РКН, вы можете настроить параметры в конфигурационном JSON-файле `rkn_config.json`, расположенном в корневой директории вашего проекта. Данная конфигурация компилируется в ассеты приложения и подгружается на старте в `RknBlocklistManager` без блокировок интерфейса.
