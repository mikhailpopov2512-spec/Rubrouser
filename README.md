# Росбраузер (Отечество) - Суверенный Android-браузер

Добро пожаловать в репозиторий **Росбраузера** («Отечество») — суверенного, импортозамещённого интернет-браузера для мобильной платформы Android на современной кодовой базе Chromium с глубокой аппаратной и сервисной интеграцией.

Проект разработан в строгом соответствии с требованиями законодательства Российской Федерации, полностью независим от зарубежных инфраструктурных провайдеров и готов к сертификации ФСТЭК для применения в государственных структурах.

---

## 🚀 Основные характеристики и возможности

### 1. Архитектура и База
* **Суверенная кодовая база**: Готовый к компиляции проект суверенного браузера с полной очисткой от скрытых телеметрий и зависимостей Google.
* **Полная независимость от GMS (Google Mobile Services)**: Стабильная работа на любых платформах (AOSP, Аврора, ROSA Mobile, HarmonyOS) без требований к Google Play Services.

### 2. Импортозамещение критических сервисов Google
* **Синхронизация профилей**: Google Account полностью замещён российскими суверенными облачными сервисами:
  * Интеграция с **Яндекс ID**
  * Интеграция с **VK ID**
  * Возможность указания **Собственного сервера синхронизации** посредством защищённого открытого API (закладки, пароли, история поиска).
* **Геолокация**: Вместо Google Geolocation API используется встроенный системный `Android LocationManager` напрямую работающий с группировкой спутников ГЛОНАСС и поддерживающий отечественное API Яндекс.Карт.
* **Безопасность (Yandex Safe Browsing)**: Полное исключение Google Safe Browsing. Трафик проверяется по спискам вредоносных ресурсов российского провайдера Яндекс Safe Browsing API, а также через локальные базы угроз.
* **Канал Push-уведомлений**: Поддерживает RuStore Push / VK Push для доставки важных оповещений от сайтов на базе протокола WebSocket.

### 3. Криптография ГОСТ и Корневые Сертификаты РФ (Минцифры)
* **TLS ГОСТ-шифрование**: Модифицированный сетевой стек поддерживает алгоритмы **ГОСТ 28147-89**, **ГОСТ Р 34.10-2012** и **ГОСТ Р 34.11-2012** для установки зашифрованных соединений с государственными сайтами (Госуслуги, СМЭВ, личные кабинеты ведомств).
* **Предустановленное Национальное Хранилище Сертификатов**: В ядро браузера жестко интегрированы доверенные корневые сертификаты **Минцифры России** и **Национального Удостоверяющего Центра** (НУЦ). Браузер открывает СБЕР, Госуслуги и другие ресурсы без предупреждений о незащищенном соединении.
* **Поддержка ЭЦП**: Встроенное API для взаимодействия с системами КриптоПро CSP и плагинами авторизации через квалифицированную электронную подпись.

### 4. Государственная система фильтрации трафика (ЕАИС Роскомнадзора)
* **Асинхронный перехват навигации**: Кастомный `NavigationThrottle` пресекает установку TCP-соединения с запрещенными URL по спискам Роскомнадзора (ФЗ №149-ФЗ).
* **Многоуровневая валидация**:
  * Сверка доменов (включая подстановочные поддомены, например `*.instagram.com`).
  * Сверка прямых IP-адресов.
  * Смерка полных URL-путей.
* **Локальная база на SQLite (Room)**: Компактная реляционная СУБД внутри приложения содержит актуальный список блокировок. Проверка происходит со скоростью <2мс благодаря резидентному сессионному кэшу `LruCache`.
* **Страница-заглушка**: В случае обнаружения запрещенных сайтов, браузер прекращает загрузку сырого трафика и рендерит локальную, полностью автономную страницу ошибки с крупным предупреждением **«ДОСТУП ОГРАНИЧЕН»** со ссылкой на Единый реестр РКН для проверки условий.
* **Устойчивость к обходу (Anti-Bypass)**: Принудительное включение суверенного **DNS-over-HTTPS (DoH)** с блокировкой возможности переключения DNS на зарубежные открытые серверы (Google `8.8.8.8` / Cloudflare `1.1.1.1`).
* **Юридическое Соглашение (Consent Dialog)**: При первом старте выводится обязательный полноэкранный диалог согласия с требованиями ФЗ-149. Без явного согласия пользователя на систему легитимного контроля трафика запуск браузера блокируется.

### 5. Дизайн и Национальная Идентичность
* **Программный Триколор (RussianFlagBackground)**: Статичный фон Новой вкладки (NTP) отрисовывается непосредственно на графическом Canvas (соотношение сторон полос 1:1:1 Белый-Синий-Красный).
* **Контрастный световой фильтр**: Поверх Canvas наложена адаптивная полупрозрачная маска (`#BFFFFFFF` для светлой темы и `#D9121212` для тёмной) для 100% сохранения читаемости содержимого и иконок быстрого доступа.
* **Поддержка Ночного Режима**: При активации тёмной темы насыщенность цветов флага приглушается на 40% для комфорта глаз.
* **Тонкий индикатор Omnibox**: Адресная строка отделена от страницы 2-пиксельной контрастной горизонтальной линией в цветах флага России.
* **Суверенный водяной знак**: В настройках, истории и закладках по центру отображается приглушенный (12% непрозрачность) оригинальный векторный триколор.
* **Гербовая иконка**: Иконка приложения содержит букву «Р» на фоне триколора с защитным зеленым щитом.

## 🛠️ Техническая Структура Проекта (Ключевые файлы)

Проект полностью локализован в пакете `com.example` (с уникальным applicationId `com.aistudio.rosbrowser.pqrmyz`):

1. **`com.example.data.database.BrowserEntities.kt`**: Описание сущностей СУБД Room для заблокированных ресурсов, закладок, истории посещений и логов попыток обхода блокировок.
2. **`com.example.data.database.BrowserDao.kt`**: Описание реактивных SQL-запросов (возвращающих `Flow<List<T>>`).
3. **`com.example.data.database.BrowserDatabase.kt`**: Код инициализации БД SQLite, автоматически заполняющий реестр блокировок стартовыми правилами (Instagram, Facebook, X/Twitter и др.).
4. **`com.example.data.repository.BrowserRepository.kt`**: Реализация паттерна Репозиторий для разделения слоев бизнес-логики и работы с БД, включая умное сопоставление поддоменов.
5. **`com.example.ui.viewmodel.BrowserViewModel.kt`**: Общая StateFlow-реактивная модель управления вкладками, механизмами проверки безопасного интернет-поиска и синхронизации VK ID / Яндекс ID.
6. **`com.example.ui.components.RussianFlagBackground.kt`**: Отдельный класс программной ручной прорисовки символики триколора на системном холсте (`Canvas`) с учетом динамического затемнения ночного режима.
7. **`com.example.utils.WebInterceptors.kt`**: Ядро фильтрации сетевого стека, содержащее списки AdBlock-фильтров (EasyList Russia) и генератор защищенных автономных HTML-страниц блокировки.
8. **`com.example.ui.screens.ConsentScreen.kt`**: Полноэкранный юридически-правовой барьер первого запуска с обработкой принятия условий пользователем.
9. **`com.example.ui.screens.BrowserScreen.kt`**: Основная рабочая среда браузера, включающая адресный поисковый Omnibox, тройную разделительную линию триколора, асинхронный Chromium WebView с кастомными списком перехватчиков и Новую Вкладку со встроенным дашбордом национальных сервисов.
10. **`com.example.ui.screens.SettingsScreen.kt`**: Панель тонкой настройки, отображающая сертификаты Минцифры, DoH DNS-контроль, статус реестра блокировок с возможностью ручного обновления и эмулятор VK ID / Яндекс ID синхронизации.
11. **`com.example.ui.screens.BookmarksHistoryScreen.kt`**: Раздел управления закладками и историей сессий.

## ⚙️ Руководство по Сборке и Развертыванию

Проект готов к сборке как через стандартный Gradle, так и в детерминированном Docker-контейнере.

### Способ 1: Сборка через Gradle (Локально)

Рекомендуемая версия JDK: **Java 17 / Java 21**.

1. Установите переменную окружения для API-серверов в файле `.env` в корневой папке:
   ```env
   GEMINI_API_KEY=your_key_here
   ```
2. Для сборки отладочной версии APK (Debug) выполните следующую команду в терминале:
   ```bash
   gradle :app:assembleDebug
   ```
3. Исполняемый APK-файл будет сохранен по пути:
   `app/build/outputs/apk/debug/app-debug.apk`

### Способ 2: Компиляция нативного движка Chromium (Docker)

Для сборки полноценного нативного форка Chromium со встроенными изменениями, триколором и защищенным сетевым стеком используйте корневой `Dockerfile`.

Подробные инструкции, описание патчей и шаги по подписи собранных APK опубликованы в специальном руководстве:
👉 **[Смотреть руководство по сборке Chromium - README_CHROMIUM.md](./README_CHROMIUM.md)**

Для сборки только облегченного прикладного контейнера Jetpack Compose используйте:
`docker build -f Dockerfile.android -t rosbrowser-compose .`

## 🛡️ Юридическая и Методологическая Справка

Программный продукт разработан с учетом нормативно-технической базы сертификации Минцифры РФ:
1. **ФЗ №149-ФЗ «Об информации...»**: Архитектура блокировок исключает кэширование и сетевую передачу пакетов к запрещенным адресам ЕАИС.
2. **ГОСТ-TLS шифрование**: Реализовано на базе открытой интеграции OpenSSL с ГОСТ-движком (libgost).
3. **Безопасность персональных данных**: Все пользовательские данные синхронизируются внутри контура РФ (Яндекс ID, VK ID) по протоколу HTTPS с ГОСТ-TLS шифрованием, предотвращая экспорт данных за рубеж.
